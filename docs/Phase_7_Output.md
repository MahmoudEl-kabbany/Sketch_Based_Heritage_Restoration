## 1. Purpose

Phase 7 is the final stage of the restoration pipeline. It handles final formatting of the geometry, visualization overlays, logging, and JSON report generation. 

---

## 2. Pipeline Overview

```
Final Paths + Metrics + Diagnostics
    │
    ├─► Sort final paths for deterministic output
    │
    ├─► Render Labeled Image
    │       └─► (Original paths + Bridges highlighted + Candidate IDs)
    │
    ├─► Render Unlabeled Overlay
    │       └─► (Clean binary mask of restored sketch)
    │
    ├─► Render Debug Visualization
    │       └─► (Shows gap detection and extracted Bézier handles)
    │
    ├─► Build Data Report
    │       └─► Compile timings, candidate stats, ASP stats, EFD diagnostics
    │
    └─► Return RestorationResult (and save report.json)
```

## 3. Post-Processing & Sorting

To ensure that labels, array indices, and visualizations are consistent across different runs, the final paths are sorted geographically (top-down, then left-right).

```python
def final_path_sort_key(p: BezierPath):
    pts = p.sample(5)
    min_y = np.min(pts[:, 1])
    min_x = np.min(pts[:, 0])
    # Group by 12-pixel vertical bands, then sort horizontally
    return (round(min_y / 12.0), min_x)

final_paths.sort(key=final_path_sort_key)
```

## 4. Visualizations

The pipeline generates three image artifacts in the output directory:

### 4.1 Labeled Restoration (`_restored.png`)
Generated by `_save_labeled_restoration()`.
- Renders original skeleton lines in **gray**.
- Renders newly added bridges and EFD closures in **green** (or high-contrast colors).
- Adds text labels indicating the `Candidate ID` and the bridging scenario.
- Returns a list of `change_logs` describing each visual change.

### 4.2 Clean Overlay (`_overlay.png`)
Generated by `_save_unlabeled_restoration_overlay()`.
- Renders the exact final topology as a clean binary mask (black lines on white background, or transparent overlay).
- This is the "production ready" output image evaluated by the `evaluate_restoration.py` framework.

### 4.3 Debug Visualization (`_debug.png`)
Generated by `_save_visualization()`.
- Shows the original raster image.
- Overlays the Bézier curve control points and handles.
- Highlights extracted endpoints with tangent vectors.

## 6. Return Object

The API entry point `restore()` returns a `RestorationResult` dataclass that contains the Python objects directly, allowing downstream code to consume the paths without parsing JSON or images.

```python
@dataclass
class RestorationResult:
    image_path: str
    original_paths: List[BezierPath]
    restored_paths: List[BezierPath]
    bridges: List[BezierSegment]
    report: dict
```

When processing multiple images, `restore_batch()` iterates sequentially and returns a `List[RestorationResult]`.
